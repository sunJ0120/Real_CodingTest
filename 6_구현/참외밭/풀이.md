## 첫 풀이 
```java
import java.lang.*;
import java.util.*;
import java.io.*;

public class Main{
    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringTokenizer st;
        int k = Integer.parseInt(br.readLine());
        int[][] length = new int[6][2];
        boolean[] isVisit = new boolean[4];  // 방향 또 나오는지 보기 위함
        for(int i = 0; i<6; i++){
            st = new StringTokenizer(br.readLine());
            length[i][0] = Integer.parseInt(st.nextToken());   // 방향
            length[i][1] = Integer.parseInt(st.nextToken());   // 길이
        }
        
        int maxLen1 = 0;  // (4,3) 중 가장 긴거
        int maxLen2 = 0;  // (2,1) 중 가장 긴거
        for(int[] len : length){
            if(len[0] == 4 || len[0] == 3){
                maxLen1 = Math.max(len[1],maxLen1);
            }else{
                maxLen2 = Math.max(len[1],maxLen2);
            }
        }
        
        // isVisit인데 또 나오는 방향 체크 * 이전 방향
        int minus = 0;
        for(int i = 0; i<6; i++){
            int dir = length[i][0];  //방향
            if(isVisit[dir]){
                minus = (length[i-1][1] * length[i][1]);
                break;
            }
            isVisit[dir] = true;
        }
        
        int answer = ((maxLen1 * maxLen2) - minus) * k;
        System.out.println(answer);
    }
}
```
- 생각해보니, 이거 시작점이 랜덤이라 만약에 이런식으로 나온다면

- 두 번 나온다는 이유로 가운데를 뽑을 수 없다.
- 그렇다면 가운데 값을 구할 방법이 없는거 같은데
- 내가 접근을 잘못한건가? 그럼 어떻게?

## 제안
- 이 문제의 경우엔, 규칙 공식이 존재한다. (이런걸 실전에서 어떻게 그냥 푸냐,)
- 작은 직사각형 찾기 : 큰 변의 인덱스 +3 위치!
- len[(maxHIdx + 3) % 6] 이다. 

- 이걸 이용해서 구하면, max 인덱스 값만 있으면 손 쉽게 구할 수 있다.

## 최종 풀이
```java
import java.lang.*;
import java.util.*;
import java.io.*;

public class Main{
    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringTokenizer st;
        int k = Integer.parseInt(br.readLine());
        int[][] length = new int[6][2];
        
        for(int i = 0; i<6; i++){
            st = new StringTokenizer(br.readLine());
            length[i][0] = Integer.parseInt(st.nextToken());   // 방향
            length[i][1] = Integer.parseInt(st.nextToken());   // 길이
        }
        
        int maxHeight = 0;  // (4,3) 중 가장 긴거
        int maxWight = 0;  // (2,1) 중 가장 긴거
        for(int[] len : length){
            if(len[0] == 4 || len[0] == 3){
                maxHeight = Math.max(len[1],maxHeight);
            }else{
                maxWight = Math.max(len[1],maxWight);
            }
        }
        
        // 인덱스 구하기
        int maxhInd = 0;
        int maxwInd = 0;
        for(int i = 0; i<6; i++){
            if(length[i][0] == 4 || length[i][0] == 3){
                if(length[i][1] == maxHeight){
                    maxhInd = i;
                }
            }else{
                if(length[i][1] == maxWight){
                    maxwInd = i;
                }
            }
        }
        
        int minusHeight = length[(maxwInd+3)%6][1];
        int minusWight = length[(maxhInd+3)%6][1];
        
        int answer = ((maxHeight * maxWight) - (minusHeight * minusWight)) * k;
        System.out.println(answer);
    }
}
```

- 방향 문제의 경우, 항상 이런식으로 크기로 나눠서 구하는 패턴이 흔히 존재한다.
- 구현 문제에서 흔히 나오는 느낌의 패턴이므로, 알아두도록 하자.

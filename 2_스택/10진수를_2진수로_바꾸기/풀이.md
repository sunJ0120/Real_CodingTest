### 내 풀이

```java
import java.util.*;
import java.lang.*;

public String solution(String s) {
    StringBuilder answer = new StringBuilder();
    int num = Integer.parseInt(s); //str to int
    int n = 0;

    //1. 가장 가까운 수 찾기
    while(Math.pow(2,n) < num){
        n++;
    }
    //2. 2진수로 변환
    for(int i = n-1; i>=0; i--){
        if(num - Math.pow(2,i) >= 0){
            answer.append(1);
            num -= Math.pow(2,i);
        }else{
            answer.append(0);
        }
    }
    return answer.toString();
}
```

사실 왜 스택을 사용하는 문제인지는 모르겠지만…

10진수 2진수 변화 메커니즘이 큰 수를 빼는걸로 이뤄지니까 가장 가깝지만 작은 수를 찾아서,
뺐을때 0보다 크면 하나 있는거니까 1을 더하고, 아니면 그 제곱은 없는거니까 0을 주는 방식으로 해결했다.

저자 출제라 프로그래머스 채점이 안되는게 상당히 불편하네….

### 참고 풀이

```java
import java.util.*;
import java.lang.*;

String solution(String s) {
    StringBuilder answer = new StringBuilder();
    int num = Integer.parseInt(s); //str to int

    //StringBuilder에 저장해서 바로 반환 + 뒤집어서 읽기
    while(num > 0){ //0일때 멈춰야 함
        answer.append(num%2);
        num /= 2;
    }
    return answer.reverse().toString();
}
```

결국 거꾸로 읽는 것이니 이것도 개념적으로 스택이라고 할 수 있긴 하다!

스택을 굳이 사용하는 것 보다는, 이렇게 StringBuilder의 reverse()를 이용하면 간결하고 효율성 좋은 풀이가 가능하다.
